## 1. 引言

### 1.1 编写目的

本文档定义「车辆碰撞虚拟仿真系统」中**碰撞子系统**的功能需求、非功能需求、验收标准及接口约束，作为开发、测试、答辩演示与结题报告的统一依据。

### 1.2 项目背景

本系统为虚拟现实课程实验项目，要求构建包含多辆可驱动车辆、车辆间及车辆与环境物体碰撞的虚拟仿真环境，并提供友好的 UI 交互与物理参数可视化能力。碰撞效果是系统核心，其中**碰撞后车辆形变**为必选项。

### 1.3 术语与缩写

| 术语 | 说明 |
|------|------|
| 刚体（Rigidbody） | 受物理引擎驱动的刚体对象 |
| 碰撞体（Collider） | 用于检测接触的碰撞几何体 |
| 恢复系数（Restitution） | 碰撞后相对速度恢复程度，0 为完全非弹性，1 为完全弹性 |
| 摩擦系数（Friction） | 接触面滑动阻力系数 |
| 形变（Deformation） | 碰撞后车身几何或外观发生的局部改变 |
| LOD | 多细节层次（Level of Detail） |

## 2. 系统概述

### 2.1 系统目标

构建一套在虚拟三维场景中运行的车辆碰撞仿真系统，使用户能够：

1. 驾驶 3–5 辆不同属性的车辆；
2. 体验车辆与车辆、车辆与环境物体之间的物理碰撞；
3. 观察**碰撞后车身形变**及连锁物理响应；
4. 通过 UI 调节关键物理参数并实时查看碰撞数据可视化。

### 2.2 系统边界

**包含：**

- 车辆驱动与物理模拟
- 碰撞检测与碰撞响应计算
- 碰撞形变表现与损伤状态管理
- 环境可碰撞物体及差异化响应
- 碰撞相关 UI 与数据可视化
- 碰撞音效与基础视觉特效（可选增强）

**不包含（本期）：**

- 真实交通法规与 AI 交通流仿真
- 高精度有限元（FEA）结构仿真
- 多人在线联机（除非小组扩展）

### 2.3 用户角色

| 角色 | 描述 |
|------|------|
| 操作者 | 通过 UI 或输入设备驾驶车辆、触发碰撞实验 |
| 观察者 | 观看碰撞过程、慢动作回放、参数曲线 |
| 开发者/答辩者 | 调节物理参数、演示系统能力、说明实现过程 |

## 3. 总体需求

### 3.1 逼真碰撞的定义（验收原则）

系统碰撞效果须同时满足以下四个维度，缺一不可：

```
逼真碰撞 = 物理响应正确 + 形变表现可见 + 多材质差异化 + 交互可观测
```

| 维度 | 最低要求 |
|------|----------|
| 物理响应 | 动量、速度、质量差异在碰撞后表现合理，无穿模、无异常弹飞 |
| 形变表现 | 碰撞后车身可见局部形变，且与碰撞强度、碰撞部位相关 |
| 材质差异 | 车辆、植物、桥梁、建筑等对象碰撞响应明显不同 |
| 交互可观测 | UI 可显示速度、力、摩擦等参数，支持调参与可视化 |

## 4. 功能需求

### 4.1 车辆管理（FR-V）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-V-01 | 系统至少包含 **3–5 辆**可独立控制的汽车 | 必须 |
| FR-V-02 | 每辆车具有独立物理属性：质量、重心高度、轴距、最大驱动力、最大制动力 | 必须 |
| FR-V-03 | 每辆车支持前进、后退、转向、制动等基本驾驶操作 | 必须 |
| FR-V-04 | 车辆间物理属性应有差异（如轿车、SUV、货车等），以保证碰撞结果可区分 | 必须 |
| FR-V-05 | 车辆状态包含：正常、受损、严重损毁（至少三档） | 必须 |
| FR-V-06 | 支持单车驾驶与多车同场景停放/行驶 | 必须 |

### 4.2 碰撞检测（FR-C）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-C-01 | 实现**车辆与车辆**之间的碰撞检测与响应 | 必须 |
| FR-C-02 | 实现**车辆与环境物体**之间的碰撞检测与响应 | 必须 |
| FR-C-03 | 环境物体至少包含：**植物、桥梁、建筑**三类 | 必须 |
| FR-C-04 | 碰撞检测应基于连续物理模拟，不允许物体相互穿透（tunneling 需处理） | 必须 |
| FR-C-05 | 支持碰撞层（Layer）或碰撞矩阵，区分不同对象类型的碰撞规则 | 必须 |
| FR-C-06 | 记录每次碰撞事件：时间戳、碰撞对象、接触点、碰撞法线、相对速度、冲量 | 必须 |
| FR-C-07 | 支持同一帧内多接触点碰撞（如侧面刮擦） | 建议 |
| FR-C-08 | 支持碰撞后二次碰撞（连锁反应，如 A 撞 B，B 撞护栏） | 必须 |

### 4.3 碰撞物理响应（FR-P）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-P-01 | 碰撞响应遵循动量守恒原则，大车撞小车时小车位移/旋转更明显 | 必须 |
| FR-P-02 | 支持配置恢复系数（Restitution），不同材质可设不同值 | 必须 |
| FR-P-03 | 支持配置静摩擦系数与动摩擦系数 | 必须 |
| FR-P-04 | 碰撞冲量应作用于接触点，产生合理的角动量变化（车辆可旋转、侧滑） | 必须 |
| FR-P-05 | 不同碰撞类型（正面、侧面、追尾）产生不同的受力与运动结果 | 必须 |
| FR-P-06 | 高速碰撞与低速碰撞的冲击强度应有明显差异 | 必须 |
| FR-P-07 | 车辆悬挂简化模拟（可选）：碰撞或颠簸时车身俯仰/侧倾 | 建议 |

### 4.4 碰撞形变（FR-D）★ 核心必选项

> **说明：** 本节为项目硬性要求。碰撞后车辆形变是区分「刚体弹球式碰撞」与「真实车辆碰撞仿真」的关键特征。

#### 4.4.1 形变总体要求

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-D-01 | 车辆碰撞后必须产生**可见的车身形变** | **必须** |
| FR-D-02 | 形变程度与碰撞强度（相对速度、冲量）正相关 | **必须** |
| FR-D-03 | 形变位置与碰撞接触区域相关（撞前脸变形前脸，侧面撞变形车门） | **必须** |
| FR-D-04 | 形变后车辆碰撞体应同步更新或近似更新，避免「外观凹了但碰撞体还是原状」 | 必须 |
| FR-D-05 | 支持累积损伤：同一车辆多次碰撞后形变加重 | 必须 |
| FR-D-06 | 严重损毁状态下车辆驾驶性能应下降（如最高速度降低、转向变差） | 建议 |

#### 4.4.2 形变实现方式（至少实现一种）

| 方案 | 描述 | 适用场景 | 优先级 |
|------|------|----------|--------|
| 方案 A：顶点形变 | 对车身 Mesh 接触区域顶点沿法线方向位移 | Unity/UE 均可，效果直观 | **推荐** |
| 方案 B：预制损伤模型切换 | 按损伤等级切换不同 Prefab（完好→轻损→重损） | 实现简单，适合答辩 | 必须可接受 |
| 方案 C：骨骼/变形目标（Blend Shape） | 预设多个形变目标，按冲量混合 | 美术可控，性能好 | 推荐 |
| 方案 D：局部缩放/凹陷 | 碰撞点附加凹陷子物体或 Decal | 快速实现 | 最低可接受 |

**验收底线：** 无论采用何种技术路线，答辩时必须能清晰展示「碰撞前 → 碰撞瞬间 → 碰撞后形变」的对比。

#### 4.4.3 形变区域划分

每辆车至少划分以下可形变区域：

| 区域 | 说明 |
|------|------|
| 前保险杠/引擎盖 | 正面碰撞主要形变区 |
| 后保险杠/后备箱 | 追尾碰撞主要形变区 |
| 左/右侧车门 | 侧面碰撞主要形变区 |
| 车顶（可选） | 翻滚或重物砸压 |

#### 4.4.4 形变参数

| 参数 | 说明 | 可调 |
|------|------|------|
| 形变阈值 | 低于该冲量不产生形变 | 是 |
| 最大形变深度 | 单区域最大凹陷距离 | 是 |
| 形变半径 | 接触点影响范围 | 是 |
| 形变衰减 | 边缘平滑过渡系数 | 是 |
| 损伤等级阈值 | 轻损/重损/报废的冲量分界 | 是 |

#### 4.4.5 形变与物理的耦合

```
碰撞发生
  → 计算冲量与接触点
  → 判断是否超过形变阈值
  → 是：更新形变网格 / 切换损伤状态 / 调整碰撞体
  → 更新车辆损伤等级
  → 触发视觉特效与音效
  → 输出碰撞数据至 UI
```

### 4.5 环境物体碰撞（FR-E）

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-E-01 | **植物**：碰撞后弯曲、倾倒或折断，对车辆阻力小 | 必须 |
| FR-E-02 | **桥梁**：桥墩/护栏为刚性或半刚性，车辆形变为主、桥体不变或轻微变化 | 必须 |
| FR-E-03 | **建筑**：墙体/立柱为刚性，高速碰撞时车辆严重形变并大幅减速 | 必须 |
| FR-E-04 | 环境物体应配置独立的恢复系数与摩擦系数 | 必须 |
| FR-E-05 | 可破坏物体（植物、护栏）与不可破坏物体（承重墙、桥墩）应区分 | 必须 |
| FR-E-06 | 环境物体被碰撞后可产生简单物理反馈（如树木倒下、护栏位移） | 建议 |

| 编号 | 需求描述 | 优先级 |

| FR-U-04 | 支持调节摩擦系数（静摩擦、动摩擦） | 必须 |
| FR-U-05 | 支持调节恢复系数 | 必须 |
| FR-U-06 | 支持调节形变相关参数（形变阈值、最大深度等） | 必须 |
| FR-U-07 | 实时显示当前车速、加速度 | 必须 |
| FR-U-08 | 实时显示最近一次碰撞的冲量/碰撞力 | 必须 |
| FR-U-09 | 提供碰撞力-时间或速度-时间曲线可视化 | 建议 |
| FR-U-10 | 提供车辆损伤状态指示（完好/轻损/重损） | 必须 |
| FR-U-11 | 支持重置场景（车辆位置、形变状态、环境物体） | 必须 |

## 6. 物理参数规格

### 6.1 车辆默认参数参考

| 车型 | 质量 (kg) | 最大速度 (km/h) | 恢复系数 | 静摩擦 | 动摩擦 |
|------|-----------|-----------------|----------|--------|--------|
| 轿车 | 1200–1500 | 120–180 | 0.15–0.25 | 0.8–1.0 | 0.6–0.8 |
| SUV | 1800–2200 | 100–150 | 0.15–0.25 | 0.7–0.9 | 0.5–0.7 |
| 货车 | 3000–8000 | 80–100 | 0.10–0.20 | 0.6–0.8 | 0.4–0.6 |

> 以上为初始默认值，须支持 UI 调节。

### 6.2 环境物体默认参数参考

| 物体类型 | 恢复系数 | 摩擦系数 | 是否可破坏 | 形变主体 |
|----------|----------|----------|------------|----------|
| 植物 | 0.05–0.15 | 0.3–0.5 | 是 | 环境物体 |
| 护栏 | 0.10–0.20 | 0.4–0.6 | 部分 | 环境/车辆 |
| 桥墩 | 0.05–0.10 | 0.5–0.7 | 否 | 车辆 |
| 建筑墙体 | 0.05–0.10 | 0.6–0.8 | 否 | 车辆 |

### 6.3 形变参数默认参考

| 参数 | 轿车建议值 | 说明 |
|------|------------|------|
| 形变阈值（冲量 N·s） | 500–1500 | 低于此值无形变 |
| 最大形变深度 (m) | 0.05–0.30 | 单区域凹陷极限 |
| 形变影响半径 (m) | 0.3–1.0 | 接触点周围影响范围 |
| 轻损阈值 | 1500–5000 | 冲量累计或单次 |
| 重损阈值 | 5000–15000 | 车辆接近报废 |

## 8. 接口需求

### 8.1 碰撞事件接口

每次碰撞应产生如下数据结构（语言无关）：

```
CollisionEvent {
    timestamp: float           // 碰撞时间
    objectA: EntityRef         // 碰撞对象 A
    objectB: EntityRef         // 碰撞对象 B
    contactPoint: Vector3      // 世界坐标接触点
    contactNormal: Vector3     // 碰撞法线
    relativeVelocity: float    // 相对速度标量
    impulse: float             // 冲量大小
    collisionType: enum        // Vehicle-Vehicle | Vehicle-Environment
}
```

### 8.2 形变模块接口

```
DeformationSystem {
  // 输入碰撞事件，输出是否发生形变
  applyDeformation(event: CollisionEvent) → DeformationResult

  // 查询车辆当前损伤状态
  getDamageLevel(vehicle: EntityRef) → DamageLevel

  // 重置车辆形变
  resetDeformation(vehicle: EntityRef) → void
}

DeformationResult {
    deformed: bool
    region: DeformRegion       // Front | Rear | Left | Right
    depth: float               // 本次形变深度
    newDamageLevel: DamageLevel
}
```

### 8.3 UI 数据接口

UI 层应能订阅以下数据（轮询或事件驱动均可）：

- 当前选中车辆的速度、加速度
- 最近一次碰撞的冲量、碰撞力
- 当前车辆损伤等级
- 全局物理参数（摩擦、恢复系数等）

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 形变实现复杂度高 | 延期或效果差 | 优先实现方案 B（损伤模型切换），再迭代方案 A |
| 高速碰撞穿模 | 仿真失真 | 启用 CCD（连续碰撞检测）；降低最大速度上限 |
| 形变与碰撞体不同步 | 视觉与物理矛盾 | 形变后更新简化 Box/Mesh Collider，或用多段碰撞体 |
| 多车性能下降 | 帧率不足 | 降低形变网格精度；远处车辆简化物理 |
| 参数难调 | 手感不真实 | 使用本文档 6.1–6.3 默认值，逐步微调 |
